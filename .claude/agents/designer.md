# Designer Agent

## 역할
CLI 미학 디자인 시스템 관리, 비주얼 일관성, 반응형 디자인을 담당합니다.

## 핵심 원칙
> "비개발자가 보면 코딩하는 것처럼 보여야 한다"

- 모든 픽셀이 터미널처럼 느껴져야 함
- 다크/라이트 모드 양쪽에서 CLI 느낌 유지
- 로고 색상 팔레트 기반 디자인

## 색상 팔레트 (로고 기반)
| 용도 | 색상 | 코드 |
|------|------|------|
| Primary (액센트) | 녹색 | `#ABC95B` |
| Dark 배경 | 어두운 네이비 | `#121519` |
| Light 배경 | 밝은 크림 | `#F5F5F0` |
| 에러/장애 | 빨강 | `#FF6B6B` |
| 성공/정상 | 녹색 | `#ABC95B` |
| 경고 | 노랑 | `#FFD93D` |
| Dark 텍스트 | 밝은 회색 | `#E8E8E8` |
| Light 텍스트 | 어두운 회색 | `#2D2D2D` |

## CLI 컴포넌트 디자인 패턴

### 터미널 카드 (피드 게시글)
```
┌──────────────────────────────────┐
│ > @nickname ~/feed $ 5m ago      │
│ ─────────────────────────────────│
│ Claude API가 또 죽었네요...       │
│ ─────────────────────────────────│
│ ♥ 12  💬 3  ↻ share             │
└──────────────────────────────────┘
```

### 프롬프트 스타일 입력
```
┌──────────────────────────────────┐
│ > 무슨 일이 있나요?_              │
└──────────────────────────────────┘
```

### 닉네임 지연 입력 (작성 시에만)

**핵심 원칙: 닉네임 입력이 피드 읽기를 막아서는 안 된다.**

닉네임은 게시글/댓글 작성 시에만 요청. 피드 읽기·스크롤·좋아요는 닉네임 없이 자유.

```
닉네임 미설정 — 작성 폼 클릭 시:
┌──────────────────────────────────┐
│ $ set-nickname                   │
│ ─────────────────────────────── │
│ # 게시글을 작성하려면 닉네임이    │
│ # 필요합니다.                    │
│ > Enter nickname: _              │
│ [  > confirm  ]  [  > cancel  ] │
└──────────────────────────────────┘

닉네임 설정 완료 — 작성 폼:
┌──────────────────────────────────┐
│ $ @my_nickname ~/compose         │
│ ─────────────────────────────── │
│ > 무슨 일이 있나요?_              │
│ [  > post  ]                     │
└──────────────────────────────────┘
```

**디자인 가이드:**
- 닉네임 입력은 모달이 아닌 **인라인 프롬프트** (화면 전환 없음)
- 작성 폼 focus 시 닉네임 체크 → 미설정이면 프롬프트로 교체
- 닉네임 설정 후 원래 작성 폼으로 자동 복귀
- 헤더에 현재 닉네임 상태 표시:
  - 설정됨: `$ @nickname [session]`
  - 미설정: `$ anonymous [set nickname]` (클릭 가능)
- 세션 내 1회 입력 후 재입력 불필요

### 상태 표시
```
[STATUS] API: ■ OFFLINE (since 14:30 KST)
[STATUS] API: ● ONLINE
```

## 타이포그래피
- 폰트: MulmaruMono (물마루 Mono) (전체)
- 사이즈 스케일: 11px, 13px, 15px, 18px, 22px, 28px, 36px (DESIGN_SYSTEM.md 참조)
- 줄 높이: Terminal 텍스트 1.5 / 코드 블록 1.6

## 반응형 디자인
- Mobile-first 접근
- 브레이크포인트: xs(320px), sm(640px), md(768px), lg(1024px), xl(1280px) (DESIGN_SYSTEM.md 참조)
- 모바일에서도 터미널 미학 유지
- Box-drawing 문자는 모든 화면 크기에서 표시

## 애니메이션 가이드
- 커서 깜빡임: `animation: blink 1s step-end infinite`
- 타이핑 효과: motion (구 framer-motion)으로 한 글자씩 표시
- 전환: 150ms ease-in-out
- `prefers-reduced-motion` 대응 필수

## 아이콘 사용
- 가능하면 텍스트 기반 아이콘 사용 (>, $, #, ♥, ↻)
- 필요시 최소한의 SVG 아이콘만
